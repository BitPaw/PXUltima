Visual C++ für Linux-Entwicklung

Einrichten Ihres Projekts für die Linux-Entwicklung

Mit dieser Workload können Sie C++-Code für Linux-Server, -Desktops und -Geräte erstellen. Sie können Ihre Verbindungen mit diesen Computern aus Visual Studio verwalten. VS kopiert Ihre Quellen automatisch und führt eine Remotekompilierung durch, und Sie können Ihre Anwendung mit dem Debugger starten. Unser Projektsystem unterstützt die Verwendung spezifischer Zielarchitekturen, ARM eingeschlossen.

Verbindungsherstellung mit Linux

Erforderliche Komponenten

Aktuell wird nur die Remotekompilierung auf dem Linux-Zielcomputer unterstützt. Es gelten keine Einschränkungen im Hinblick auf die Linux-Distributionen, aber es müssen einige Tools vorhanden sein. Insbesondere werden openssh-server, g++, gdb und gdbserver benötigt. Verwenden Sie Ihren bevorzugten Paket-Manager, um diese Tools zu installieren, z. B. auf Debian-basierten Systemen: sudo apt-get install openssh-server g++ gdb gdbserver

Erste Verbindung

Wenn Sie einen Linux-Computer zum ersten Mal als Ziel verwenden, werden Sie zur Eingabe von Verbindungsinformationen aufgefordert. Diese Aufforderung wird durch das Erstellen des Projekts ausgelöst.

Hinzufügen und Entfernen von Verbindungen

Um eine neue Verbindung hinzuzufügen, wechseln Sie zu "Extras" > "Optionen", und suchen Sie nach "Verbindung". Der Verbindungs-Manager befindet sich unter "Plattformübergreifend". Hier können Sie Verbindungen hinzufügen und entfernen.

Wenn Sie die von einem Projekt genutzte Verbindung ändern möchten, aktualisieren Sie in den allgemeinen Einstellungen für die Projekteigenschaften die Option für den Remotebuildcomputer.

Projekteigenschaften

Alle Optionen, die zum Steuern der C++-Kompilierung erforderlich sind, werden auf den Eigenschaftenseiten des Projekts verfügbar gemacht. Wir werden hier auf einige Besonderheiten im Hinblick auf die Funktionsweise für Linux eingehen. Zunächst werden Sie feststellen, dass der Remotestamm in den allgemeinen Einstellungen standardmäßig auf "~/projects/" festgelegt ist und das Remoteprojektverzeichnis mit dem Projektnamen an diesem Speicherort übereinstimmt.

Anhand der allgemeinen Einstellungen für das Projekt können Sie erkennen, wie die Ausgabe- und Zwischenverzeichnisse konfiguriert wurden. Außerdem sehen Sie, dass dieses Projekt als Anwendung konfiguriert wurde – aus diesem Grund befindet sich unsere ausführbare Datei unter "bin/x64/Debug/" und trägt den Namen "ConsoleApplication1.out". Beachten Sie, dass als Konfigurationstypen auch statische und dynamische Bibliotheken unterstützt werden.

Fügen Sie zusätzliche Bibliotheksabhängigkeiten auf der Eigenschaftenseite "Linker" > "Eingabe" hinzu.

Sie können zusätzliche Befehle an den Debugger übergeben, die vor dem Start ausgeführt werden – um beispielsweise grafische Apps auf dem Linux-Remotecomputer zu starten.

Sie können außerdem Ereignisse zur Ausführung nach der Kompilierung senden, um das Remoteverhalten zu steuern – wie im vorliegenden Beispiel, bei dem ein GPIO-Pin exportiert wird, bei dessen Verwendung die ausführbare Datei nicht als Superuser ausgeführt werden muss.
